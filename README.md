# Zerobus IoT Real-Time

Real-time IoT monitoring dashboard using Databricks Zerobus, Delta Tables, and FastAPI.

Simulators run locally on any machine and send IoT events to a Delta table. A FastAPI + vanilla JS dashboard deployed as a Databricks App shows the data updating in real-time via WebSocket.

## Setup

### Step 1 — Databricks workspace

Import `setup_workspace.py` as a notebook into any Databricks workspace and run it cell by cell.

It will:
- Detect your environment automatically (user, workspace URL, warehouse)
- Create catalog, schema, and table (or use an existing catalog if you don't have permission to create one)
- Deploy the dashboard app (FastAPI + vanilla JS)
- Generate OAuth credentials for the simulator
- Show the Client ID and Secret to copy (cleared on the next cell)

### Step 2 — Local machine

```bash
python setup_local.py
```

It will:
- Install dependencies (`requests`, `databricks-zerobus-ingest-sdk`)
- Ask for workspace URL, catalog, schema, table, warehouse ID
- Ask for the SP credentials from Step 1 (hidden input)
- Test the connection
- Save everything to `config.json`

### Step 3 — Run the simulator

Pick one:

```bash
# SQL REST — simple, works everywhere
python iot_simulator_rest.py

# Zerobus gRPC — fast, direct-to-Delta
python iot_device_simulator.py
```

Then open the dashboard URL from Step 1 to see data arriving in real-time.

## Files

| File | Runs on | What it does |
|---|---|---|
| `setup_workspace.py` | Databricks | Creates table, deploys dashboard app, generates credentials |
| `setup_local.py` | Laptop | Installs deps, configures simulators, saves `config.json` |
| `iot_simulator_rest.py` | Laptop | Sends events via SQL REST API |
| `iot_device_simulator.py` | Laptop | Sends events via Zerobus gRPC SDK |
| `config.json` | Laptop | Generated by `setup_local.py`, read by simulators |

## Simulators

| | SQL REST | Zerobus gRPC |
|---|---|---|
| **Install** | `pip install requests` | `pip install databricks-zerobus-ingest-sdk` |
| **Protocol** | HTTP → SQL INSERT | gRPC direct to Delta |
| **Latency** | ~200-500ms | ~2-50ms |
| **Best for** | Quick testing, any environment | Low-latency ingestion |

## Parameters

Override via environment variables:

| Variable | Default | Description |
|---|---|---|
| `NUM_DEVICES` | 5 | Number of simulated sensors |
| `EVENTS_PER_SECOND` | 2 | Event generation rate |
| `ANOMALY_RATE` | 0.1 | Probability of anomalous readings (10%) |

## Dashboard Features

- Real-time updates via WebSocket (every 3 seconds)
- Temperature timeline and events-per-minute charts (SVG)
- Live event stream table with anomaly highlighting
- Alerts tab with severity breakdown (CRITICAL / WARNING / INFO)
- Device health cards with OK/ALERT status
- Pause/Resume button that stops server-side polling (saves warehouse cost)
